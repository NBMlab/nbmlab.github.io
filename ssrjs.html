SSR.JS — Server Side Rendered JavaScript (GitHub/Source)

Modern JavaScript library to auto-sync state and UI, which builds JS-model based on pre-rendered HTML, rather than additional JSON-request

How it works

Just define model structure, and JS-model will be populated with data from HTML

<!DOCTYPE html>
<html>
<head></head>
<body>

    <span data-model="user.name">John Watson</span>
    <span data-model="user.balance">0</span>

    <script src="https://cdn.jsdelivr.net/gh/ssrjs/ssr.js@master/ssr.js"></script>

    <script>
    const user = new Model('user', {
        name: 'string',
        balance: 'number',
    });
    </script>

</body>
</html>

Now, global object "user" can be accessed and managed (UI will be instantly synced):

>>> user.name
'John'
>>> user.balance = 10
10

<script>
    const user = new Model('user', {
            name: 'string',
            balance: 'number',
        }, {
            changeBalance(amount) {
                this.balance += amount
            }
        }
    );
</script>

<form onsubmit="products.setPrice(this, event, os.value, Number(price.value))">
    <b>Set new price</b> $<input type="text" name="price" required>
    to all
    <select name="os">
        <option value="IOS" selected>IOS phones</option>
        <option value="Android">Android phones</option>
    </select>
    <button>Update prices</button>
</form>

<script>
const products = new List('product', {
        id: 'number',
        os: 'string',
        name: 'string',
        price: 'number',
    }, {
        setPrice(form, e, os, price) {
            form.price.value='';
            e.preventDefault();
            products.select({os: os}).update({price: price});
        }
    }
);
</script>
